Task OOP

1. Створив порожній проект в IntelliJ



2. Створив клас "Співробітник" з полями: ПІБ, посада, email, телефон, вік.



3. Конструктор класу заповнюває ці поля під час створення об'єкта. 
   Забезпечив інкапсуляцію внутрішніх властивостей класу.



4. Створив два класи з однаковим ім'ям SameName. (Використовуючи пакети)



5. Створив клас Car з публічним методом start. Метод start викликає у собі методи:

- startElectricity()

- startCommand()

  - startFuelSystem()
  
  Перелічені методи, є внутрішньою (прихованою) частиною комплексного методу start. 
  Вони по суті відображають внутрішню поведінку класу Car 
  і не повинні використовуватися за межами даного класу.
